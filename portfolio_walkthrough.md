# Portfolio Walkthrough

## Story this portfolio tells

This portfolio is built around a career bridge: moving from operations and process improvement into enterprise automation, AI enablement, data analysis, and solution architecture.

The projects are intentionally practical. They show how a business-facing professional can create value before becoming a full software engineer or machine learning engineer.

## Project 01: Power Platform Solution Design

Shows how to document a workflow architecture using SharePoint/List-style data structures, Power Automate-style event logic, UX controls, Dev/Prod deployment thinking, and governance checks.

Best for showing readiness for:

- Automation Analyst
- Business Systems Analyst
- Solution Architecture Analyst
- Power Platform Analyst

## Project 02: Teams Backfill Recovery

Shows a data quality recovery pattern: missing tracker rows are repaired by joining against a synthetic Teams-style message export, normalizing HTML content, deriving sender and manager fields, and producing a recovered dataset.

Best for showing readiness for:

- Data Analyst
- Automation Analyst
- AI Analyst bridge role
- Business Systems Analyst

## Project 03: Ticket Classification and Reporting

Shows deterministic classification logic with rule precedence. This is useful when reporting metrics need to be stable, auditable, and explainable.

Best for showing readiness for:

- Reporting Analyst
- AI Analyst bridge role
- Data Quality Analyst
- Operational Analytics Analyst

## Project 04: Responsible AI Assistant Guardrails

Shows a lightweight evaluation framework for an AI assistant. It includes prompt test cases, a risk register, and an evaluation script that flags potential issues such as unsupported claims and missing guardrail language.

Best for showing readiness for:

- AI Governance Analyst
- AI Project Coordinator
- Responsible AI Analyst
- AI Product Analyst

## Project 05: Automation Use Case Intake

Shows a scoring model to prioritize automation requests based on business impact, manual effort, risk, data readiness, complexity, and stakeholder alignment.

Best for showing readiness for:

- Product Analyst
- Business Analyst
- Automation Analyst
- Solution Delivery Coordinator

## Suggested 60-second walkthrough for a leader

"This portfolio is designed to show how I approach automation and AI use cases from both the business and technical sides. I created synthetic examples that mirror the type of work I have been building toward: workflow design, data recovery, classification logic, responsible AI guardrails, and automation prioritization. I intentionally included documentation, code, data samples, and governance artifacts because I want to grow into roles that connect business needs with scalable technology delivery."
