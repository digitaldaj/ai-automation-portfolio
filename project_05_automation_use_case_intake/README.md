# Project 05: Automation Use Case Intake

## Purpose

This project demonstrates how automation opportunities can be evaluated and prioritized before build work begins.

## Business problem

Teams often have many ideas for automation, but limited technical capacity. A structured intake model helps prioritize work based on business impact, readiness, risk, and complexity.

## Solution approach

1. Capture use cases with consistent fields.
2. Score each request across business and technical dimensions.
3. Prioritize high-value and feasible opportunities.
4. Identify low-readiness ideas that need more discovery.

## Run the demo

```bash
python3 project_05_automation_use_case_intake/src/prioritize_use_cases.py
```
