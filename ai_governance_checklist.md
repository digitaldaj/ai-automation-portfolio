# AI Governance Checklist

## Use case approval

- [ ] Business owner identified.
- [ ] Intended users defined.
- [ ] Approved data sources documented.
- [ ] Sensitive topics identified.
- [ ] Human escalation path defined.

## Testing

- [ ] Prompt test cases created.
- [ ] Ambiguous prompts tested.
- [ ] Missing-source behavior tested.
- [ ] High-risk topic behavior tested.
- [ ] Overcommitment and guarantee language tested.

## Deployment readiness

- [ ] Limitations documented.
- [ ] User instructions written.
- [ ] Feedback channel created.
- [ ] Monitoring cadence defined.
- [ ] Content owner assigned.

## Continuous improvement

- [ ] Review failed responses.
- [ ] Update knowledge sources.
- [ ] Add new test cases for new risks.
- [ ] Track user adoption and recurring issues.
