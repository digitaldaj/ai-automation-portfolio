# Confidentiality and Public-Safe Portfolio Rules

This repository is designed to be safe for a public GitHub profile.

## Included

- Synthetic data
- Generic workflow names
- Public-safe architecture patterns
- Generic automation examples
- Documentation templates
- Python scripts that demonstrate logic
- SQL-style examples using sample fields

## Not included

- Customer names or identifiers
- Policy, account, claim, or transaction details
- Internal URLs or system names
- Credentials, tokens, API keys, or connection strings
- Proprietary workflow exports
- Screenshots from internal tools
- Company-owned documentation

## Review checklist before publishing

- Search the repository for real names, phone numbers, emails, policy numbers, claim numbers, ticket numbers, and URLs.
- Keep examples generic and synthetic.
- Do not upload raw exports from workplace systems.
- Do not upload Power Automate exports from internal environments.
- Use screenshots only if they are rebuilt with synthetic data.
